Examples

await using var scope = await TransformFileScope.CreateAsync(
    myProjectFile,
    content => content.Replace("1.0.0", "2.0.0"));
// File now contains "2.0.0". Original is restored when scope is disposed.

Remarks

On creation the file's current content is captured. The supplied transform is applied and written back immediately. On disposal the original content is restored, or — if the file did not exist before the scope was opened — the file is deleted.

This is useful during an operation when a file (e.g. a project file or a config file) needs to be temporarily modified for a specific step without permanently altering it.

Call CancelRestore() before disposing to commit the transformation instead of rolling it back.

Remarks

The scope still restores to the original content (captured at creation time), not the content at the point of the last Add call. Multiple calls build up transformations; disposal always unwinds them all at once.

If CancelRestore() has been called, the transformation is skipped and the scope is returned unchanged.

Dispose()

Synchronously disposes the scope and restores the file to its original state.

public void Dispose()

Remarks

  • If CancelRestore() was called, the file is left as-is.
  • If the file did not exist when the scope was created, it is deleted.
  • Otherwise the original content is written back.
  • Calling this method more than once is safe — subsequent calls are no-ops.

Prefer DisposeAsync() (via await using) where possible so the restore write does not block the calling thread.

DisposeAsync()

Asynchronously disposes the scope and restores the file to its original state.

public ValueTask DisposeAsync()

Returns

ValueTask

Remarks

  • If CancelRestore() was called, the file is left as-is.
  • If the file did not exist when the scope was created, it is deleted.
  • Otherwise the original content is written back.
  • Calling this method more than once is safe — subsequent calls are no-ops.

No cancellation token is accepted here: we prefer to wait for the write to complete so the file is not left in a partially-written state.
